# I18N y localización

## Objetivo

La app debe ser bilingüe desde el inicio.

Idiomas iniciales:

- Español: `es-ES`
- Catalán: `ca-ES`

## Selección de idioma

El idioma se elige en el onboarding inicial.

Primera pantalla:

```text
Elige idioma / Tria idioma
[Español]
[Català]
```

El idioma se puede cambiar después desde ajustes.

## Qué debe traducirse

- Textos de UI.
- Botones.
- Enunciados de ejercicios.
- Pistas.
- Textos hablados.
- Centro de ayuda.
- Recompensas.
- Ajustes.
- Mensajes de feedback.
- Etiquetas de objetos visuales.
- Singular/plural de objetos.
- Nombres de mundos y niveles.

## AppLanguage

Modelo recomendado:

```dart
enum AppLanguage {
  spanish,
  catalan,
}

extension AppLanguageX on AppLanguage {
  String get localeCode {
    switch (this) {
      case AppLanguage.spanish:
        return 'es-ES';
      case AppLanguage.catalan:
        return 'ca-ES';
    }
  }

  String get displayName {
    switch (this) {
      case AppLanguage.spanish:
        return 'Español';
      case AppLanguage.catalan:
        return 'Català';
    }
  }
}
```

## LocalizedText

Modelo recomendado:

```dart
class LocalizedText {
  final Map<String, String> values;

  const LocalizedText(this.values);

  String get(String languageCode) {
    return values[languageCode] ?? values['es-ES'] ?? '';
  }
}
```

## Textos visibles y hablados

Separar siempre.

Ejemplo:

```dart
class ExerciseText {
  final LocalizedText visibleQuestion;
  final LocalizedText spokenQuestion;
  final LocalizedText visibleHint;
  final LocalizedText spokenHint;
}
```

## Ejemplo de plantilla de ejercicio

```json
{
  "id": "character_gets_more_items",
  "type": "addition",
  "texts": {
    "es-ES": {
      "visible": "{characterName} tiene {a} {itemPlural} y recibe {b} más. ¿Cuántos tiene?",
      "spoken": "{characterName} tiene {aWords} {itemPlural} y recibe {bWords} más. ¿Cuántos {itemPlural} tiene ahora?",
      "hint": "Cuenta primero {a}. Luego añade {b} más.",
      "spokenHint": "Cuenta primero {aWords}. Luego añade {bWords} más."
    },
    "ca-ES": {
      "visible": "{characterName} té {a} {itemPlural} i en rep {b} més. Quants en té?",
      "spoken": "{characterName} té {aWords} {itemPlural} i en rep {bWords} més. Quants {itemPlural} té ara?",
      "hint": "Compta primer {a}. Després afegeix-ne {b} més.",
      "spokenHint": "Compta primer {aWords}. Després afegeix-ne {bWords} més."
    }
  }
}
```

## Números en palabras

Necesario para TTS:

```text
1 → uno / un / una
2 → dos
12 → doce
17 → diecisiete
```

En catalán:

```text
1 → un / una
2 → dos / dues
12 → dotze
17 → disset
```

Crear helper:

```text
number_to_words_es.dart
number_to_words_ca.dart
```

Tener cuidado con género:

- un corazón.
- una estrella.
- un cor.
- una estrella.

## TTS

El TTS debe usar:

- `es-ES` si idioma español.
- `ca-ES` si idioma catalán.

## Archivos de datos

Opciones:

### Opción A: archivo único

```text
assets/data/localization.json
```

### Opción B: archivos separados

```text
assets/data/i18n/ui_es.json
assets/data/i18n/ui_ca.json
assets/data/i18n/help_topics_es.json
assets/data/i18n/help_topics_ca.json
```

Para MVP, un archivo único puede ser más simple.

## Reglas para agentes

- No hardcodear textos finales en widgets.
- Todo texto nuevo debe añadirse en el sistema de localización.
- Toda plantilla de ejercicio debe tener `es-ES` y `ca-ES`.
- Todo tema de ayuda debe tener `es-ES` y `ca-ES`.
- Todo texto hablado debe revisarse separado del visible.
